# Web UI Components

## Objetivo

Criar e ajustar interfaces web mantendo consistência, responsividade e acessibilidade.

## Regras

- Reutilizar componentes existentes antes de criar novos.
- Preservar padrões visuais do projeto.
- Evitar mudanças globais de CSS sem necessidade.
- Pensar em mobile first.
- Garantir estados de hover, focus, loading e disabled quando necessário.
- Manter HTML semântico.
- Evitar animações pesadas.
- Manter contraste e legibilidade.

## Checklist

- Responsivo em mobile, tablet e desktop.
- Espaçamento consistente.
- Hierarquia visual clara.
- Botões acessíveis.
- Formulários com labels.
- Imagens com alt text.
- Componentes reaproveitáveis.
- Código simples e fácil de manter.

## Evitar

- Duplicar componentes quase iguais.
- Criar CSS global para ajuste local.
- Quebrar layout existente.
- Alterar identidade visual sem pedido.
