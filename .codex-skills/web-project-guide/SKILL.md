# Web Project Guide

## Objetivo

Orientar o Codex a trabalhar em projetos web de forma segura, organizada e econômica, respeitando a estrutura real do repositório.

## Regras principais

- Antes de editar, entender o objetivo da tarefa.
- Identificar framework, scripts disponíveis e estrutura de pastas.
- Ler apenas arquivos relevantes.
- Preservar padrões existentes do projeto.
- Evitar refatorações grandes sem pedido explícito.
- Preferir alterações pequenas, testáveis e fáceis de revisar.
- Não alterar design, texto, rotas ou regras de negócio sem necessidade.
- Sempre explicar claramente o que foi feito.

## Fluxo recomendado

1. Verificar `package.json`, estrutura principal e scripts disponíveis, se necessário.
2. Identificar a página, componente, rota, serviço ou estilo envolvido.
3. Abrir somente os arquivos relacionados.
4. Fazer a menor alteração segura.
5. Rodar validação disponível: build, lint, typecheck ou testes.
6. Informar:
   - arquivos lidos;
   - arquivos alterados;
   - motivo das alterações;
   - comandos executados;
   - próximos cuidados.

## Quando combinar com outras skills

- Para economizar contexto: usar `web-token-saver`.
- Para erro de build: usar `web-build-debugger`.
- Para performance: usar `web-performance-auditor`.
- Para SEO: usar `web-seo-content`.
- Para layout/componentes: usar `web-ui-components`.
- Para segurança: usar `web-security-review`.
- Para afiliados/produtos: usar `web-affiliate-products`.
- Para artigos/notícias/blog: usar `web-content-publisher`.
