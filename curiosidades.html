Quando o Task Manager cabia em apenas 80 KB

A primeira versão do Task Manager mostra como software leve, engenharia cuidadosa e foco no usuário continuam atuais.

Encontre Aqui Tech 19 abr. 2026 Curiosidade histórica

Hoje, quando se fala em software, é comum encontrar aplicações cada vez mais pesadas, cheias de camadas, recursos extras e consumo elevado de memória. Mas uma lembrança curiosa da história do Windows mostra que nem sempre foi assim. Segundo um relato resgatado recentemente, a primeira versão do Task Manager criada por Dave Plummer tinha apenas 80 KB, um tamanho extremamente pequeno até para os padrões mais modestos da atualidade.

A explicação para isso está no contexto da época. Nos anos 90, o hardware era muito mais limitado, e uma ferramenta pensada para socorrer o sistema em momentos de travamento precisava continuar leve, rápida e confiável. Em outras palavras, o Task Manager não podia se tornar mais um problema dentro de uma máquina já sobrecarregada.

O projeto também chamava atenção pelas escolhas inteligentes de engenharia. Em vez de apenas verificar se outra instância do programa já estava aberta, o utilitário enviava uma mensagem interna para confirmar se essa instância ainda estava respondendo. Se não houvesse resposta, ele entendia que o processo travara e abria uma nova janela para ajudar o usuário a retomar o controle do PC.

Além disso, Plummer adotou várias estratégias para economizar recursos. Algumas informações mais usadas eram carregadas uma única vez, enquanto funções menos frequentes só eram ativadas quando realmente necessárias. O mesmo vale para a leitura da árvore de processos, feita de forma mais direta para evitar chamadas excessivas ao sistema. O objetivo era simples: fazer mais com menos.

Essa história vai além da nostalgia. Ela mostra como limitações técnicas podem estimular decisões mais cuidadosas no desenvolvimento de software. Em uma era marcada por máquinas muito mais potentes, o antigo Task Manager ainda serve como lembrete de que desempenho, eficiência e experiência do usuário continuam sendo pilares importantes para qualquer aplicação bem construída.

Impressoras 3D da Bambu ganham mistura de cores com filamentos

Atualização permite simular tons e gradientes combinando dois ou três filamentos em uma mesma impressão.

A Bambu Lab atualizou seu ecossistema de impressão 3D com recursos capazes de prever combinações de cores a partir de dois ou três filamentos. A ideia lembra a mistura visual usada em impressão tradicional: pontos e camadas diferentes criam a percepção de novos tons sem exigir um filamento específico para cada cor.

Para makers e criadores, esse tipo de recurso amplia as possibilidades de acabamento sem tornar o fluxo de trabalho tão complexo. Também mostra como software e calibração estão virando diferenciais importantes em impressoras 3D, especialmente quando o objetivo é entregar peças mais expressivas sem elevar demais o custo do material.
